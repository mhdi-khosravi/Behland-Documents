# 📚 مرکز مستندات به‌لند  
### مخزن رسمی تمامی داکیومنت‌های پروژه BehLand

[**English Version**](./README.md)

---

## 🔍 معرفی
این مخزن به‌عنوان **مرجع اصلی و رسمی تمام مستندات پروژه به‌لند** ایجاد شده است.  
تمامی اسناد پروژه به‌صورت عمومی در اینجا منتشر می‌شوند تا **شفافیت، اعتماد، و امکان بررسی مستقل (DYOR)** برای همه فراهم باشد.

به‌لند یک اکوسیستم آموزشی بازی‌محور روی **شبکه TON** است که بر **یادگیری واقعی، پیشرفت مهارتی قابل‌اعتبار و ایجاد ارزش بلندمدت** تمرکز دارد.

---

## 📂 ساختار مخزن
هر فولدر در این ریپازیتوری نماینده‌ی یک بخش مشخص از اکوسیستم به‌لند است.  
با کلیک روی هر فولدر می‌توانید فایل‌ها و README مربوط به همان بخش را مشاهده کنید.

### بخش‌های اصلی شامل:
- **LitePaper** – چشم‌انداز، فلسفه، کاربرد توکن، مدل آموزشی و نقشه راه  
- **MVP** – برنامه حداقل محصول قابل ارائه، فازهای توسعه محصول و مسیر تأیید اولیه  
- **Tokenomics** – توزیع BEHT، کاربردها و ساختار اقتصادی  
- **سیستم آموزشی** – مسیرهای یادگیری، سطوح VIP، مکانیزم NTT و NFT  
- **امنیت و شفافیت** – ساختار کیف پول‌ها، مالتی‌ساین، وضعیت بازبینی امنیتی و افشاها  
- **PitchDeck** – ارائه‌های رسمی سرمایه‌گذاری (فارسی / انگلیسی)  
- **برند و رسانه** – لوگوها، هویت بصری و فایل‌های رسمی  

این ساختار به هر کاربر اجازه می‌دهد عملکرد پروژه را به‌صورت مستقل بررسی کند.

## 🚀 مستندات MVP

سند MVP به‌لند توضیح می‌دهد که نسخه‌های اولیه محصول چگونه باید مرحله‌به‌مرحله ساخته شوند تا ایده اصلی پروژه برای کاربر، ناظر اولیه و تیم بررسی‌کننده قابل فهم باشد.

این سند شامل موارد زیر است:
- MVP چیست؛
- چرا به‌لند قبل از نسخه کامل به MVP نیاز دارد؛
- سطح ۱: MVP استاندارد؛
- سطح ۲: MVP بهینه؛
- سطح ۳: MVP تکمیلی؛
- محدوده شبکه اجتماعی، کیف پول، NTT، BEHT، ریوارد، Light Relay، DAO و هوش مصنوعی همراه؛
- وضعیت فعلی MVP و هدف دریافت تأیید اولیه.

مطالعه سند MVP:

- [برنامه توسعه MVP به‌لند](./MVP/README_FA.md)
